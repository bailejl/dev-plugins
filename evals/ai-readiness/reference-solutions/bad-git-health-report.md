### Major

#### 2. Long-Lived Stale Branches

**Evidence**: `git branch -a --sort=-committerdate` reveals:

```
feature/old-thing        (last commit: 60+ days ago)
feature/abandoned-refactor (last commit: 90+ days ago)
```

These branches have not been updated in months and are significantly diverged from main.

**Impact**: Increases merge conflict risk. Signals incomplete work and abandoned features. Makes the branch list noisy and hard to navigate.

**Recommendation**:
- Delete merged or abandoned branches: `git branch -d feature/old-thing`
- Set a branch age policy (e.g., branches older than 30 days without activity should be reviewed and either merged, rebased, or deleted)
- Use `git branch --merged` to find branches safe to delete

#### 3. Environment Branches Diverged from Main

**Evidence**: `git log --oneline main..dev`, `git log --oneline main..staging`, `git log --oneline main..production` show diverged commits:

```
dev:        2 commits ahead of main, 3 behind
staging:    1 commit ahead, 5 behind
production: 2 commits ahead, 4 behind
```

Each environment branch has unique commits not in main, and all are behind main.

**Impact**: Deploying from diverged environment branches means:
- Different code running in each environment
- Hotfixes in production not propagated to dev/staging
- No clear "source of truth" branch
- Merge conflicts accumulating over time

**Recommendation**:
- Adopt a trunk-based development model where main is always deployable
- Use tags or release branches for versioning, not environment branches
- Deploy from main to all environments, using configuration (not code branches) for environment differences

#### 4. Large Binary File in Repository

**Evidence**: `git rev-list --objects --all | git cat-file --batch-check` reveals:

```
assets/logo.psd  5.0 MB
```

A 5MB PSD file is committed directly to the git repository.

**Impact**:
- Bloats repository size and clone time for every developer
- Binary files cannot be diffed or merged
- Every change to this file stores a full copy in git history

**Recommendation**:
- Remove the binary file: `git filter-branch` or BFG Repo Cleaner
- Add `*.psd` to `.gitignore`
- Use Git LFS for large binary assets: `git lfs track "*.psd"`
- Or store design assets in a separate system (Figma, S3, etc.)
